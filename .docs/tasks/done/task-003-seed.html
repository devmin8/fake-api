Backlog ticket · task-003

Seed data

A faker-powered seed that fills the DB with believable users (incl. bots), posts, comments, tags, follows, and likes — so every read endpoint built next has real relational data to return.

①

Scope

  • src/db/seed.ts using @faker-js/faker. Script db:seed.
  • Seed: ~20 real users + a handful of is_bot=true bot users (used later by the firehose); a fixed, known test user whose credentials match testEmail/testPassword in the Bruno env (so login tests work).
  • ~15 tags (name + slug); ~80 posts across users with random tags, some with image_url placeholders; threaded comments; a spread of likes, follows, bookmarks. Keep denormalized counts consistent with the rows created.
  • Idempotent-ish: a --fresh path that wipes + reseeds is fine; document whichever you choose.
②

Acceptance

  • bun run db:seed populates all tables without FK errors.
  • The known test user exists and its password_hash verifies against the Bruno test password (validated for real in task-004).
  • Denormalized counts match actual child rows (spot-check one post).
  • At least a few posts have image_url set.
③

Bruno tests

No endpoints yet. The deliverable that unblocks Bruno is the known test user — its credentials must equal the local environment's testEmail/testPassword so task-004's login test authenticates.
