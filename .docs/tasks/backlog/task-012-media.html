Backlog ticket · task-012

Media uploads

Multipart image upload that validates mime + size, stores under ./data/uploads/, records a media row, and returns a { url } usable as a post's image_url.

①

Scope

  • src/routes/media.ts:
    • POST /api/media AUTH — multipart; validate images-only mime + max size; write ./data/uploads/<ulid>.<ext>; insert media row; return { url }.
    • GET /api/media/:file PUBLIC — serve the file with the right content-type.
  • Confirm POST /posts accepts the returned URL as image_url.
②

Acceptance

  • Uploading a small PNG/JPEG returns { url } and writes the file + row.
  • Non-image mime → 422; oversize → 422 (or 413, document).
  • GET /media/:file serves the bytes with correct content-type.
  • Creating a post with that URL renders image_url on read.
  • Upload 401 unauthenticated.
③

Bruno tests — 09-media/

Auth required. Use a body:multipart-form request with a small fixture image checked into bruno/09-media/fixtures/. Assert { url }, capture it, then GET it back (200 + image content-type). Create a post using the URL and confirm it reads back. Add the 401 + bad-mime cases.
