Project skeleton

A runnable Bun + Elysia app that boots, serves /health, exposes Swagger at /docs, applies credentialed CORS, and returns errors in the standard envelope. Plus the Bruno collection skeleton every later task plugs into.

①

Scope

  • package.json, tsconfig.json, Bun + Elysia deps. Scripts: dev, start.
  • .env.example (all vars from the design's Config section) + src/config.ts (typed env loader with sane defaults).
  • src/index.ts — Elysia app, listen on PORT, mount plugins + routes.
  • src/plugins/cors.ts — credentialed CORS, single origin from CORS_ORIGIN (echo exact origin, Allow-Credentials: true, never *).
  • src/lib/errors.ts — envelope { error: { code, message, details? } } and the code→status map (VALIDATION_ERROR 422 · UNAUTHENTICATED 401 · FORBIDDEN 403 · NOT_FOUND 404 · CONFLICT 409 · RATE_LIMITED 429). Wire as Elysia onError.
  • @elysiajs/swagger at /docs.
  • GET /health → { status: "ok" }.
  • bruno/ collection: bruno.json, collection.bru (vars + the auth/CSRF pre-request from the testing plan — harmless with no creds yet), environments/local.bru (baseUrl, testEmail, testPassword), and the 01-health/ folder.
②

Acceptance

  • bun run dev starts and logs the listen URL.
  • GET /health returns { status: "ok" }.
  • /docs renders Swagger UI.
  • A thrown/unknown-route error returns the error envelope, not a raw stack.
  • Bruno: bru run 01-health --env local passes (status 200 + body shape).
③

Bruno tests

Follow bruno-testing-plan.html. This task creates collection.bru (with the collection-level auth/CSRF pre-request) and the local environment. Auth vars stay empty until task-004 — fine, the script no-ops when they're unset. Folder 01-health asserts /health returns 200 and status = ok.
