Pin GitHub Actions to commit SHAs

Supply-chain hardening for .github/workflows/publish-image.yml. Today the workflow references third-party actions by mutable version tags (@v4, @v6, …). Pin each to an immutable commit SHA so a compromised or retagged action can't run with our packages: write token.

①

Why this is parked, not done

  • The publish workflow is manually triggered on a personal repo, so the exposure window is small — this is genuine hardening, not a live hole.
  • A tag like @v6 is a mutable pointer: whoever controls the action repo can move it to new code, which would then run inside our CI job with the GITHUB_TOKEN. A full commit SHA is immutable and can't change under us. (This is the class of attack behind the tj-actions/changed-files incident.)
  • The cost: pinned SHAs don't auto-update, so they must be kept fresh — otherwise we drift onto stale, unpatched actions. So this ticket pairs the pins with Dependabot, which is the part that makes pinning sustainable.
②

Scope

  • Rewrite each uses: in publish-image.yml to owner/action@<40-char-sha> # vX — SHA for immutability, the trailing # vX comment for human readability and so Dependabot can bump both together.
  • Add .github/dependabot.yml with the github-actions ecosystem (e.g. weekly) so the pinned SHAs get PR'd forward automatically.
  • Re-run the workflow once to confirm nothing broke after pinning.
