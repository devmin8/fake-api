Database & schema

SQLite (bun:sqlite) wired to Drizzle, the full relational schema defined, and migrations that run on boot so the app creates ./data/app.db with every table.

①

Scope

  • src/db/index.ts — bun:sqlite connection (path from DB_PATH) + Drizzle client, PRAGMA foreign_keys = ON.
  • src/db/schema.ts — all tables + relations from the design: users, posts, comments, likes (polymorphic), follows, bookmarks, tags, post_tags, notifications, refresh_tokens, media.
    • ULID primary keys (time-sortable). ISO-8601 UTC timestamps.
    • Denormalized counts on posts (like_count, comment_count, view_count) and comments (like_count).
    • Uniqueness: users.username/email, tags.name/slug, likes(user_id,target_type,target_id); composite PKs for follows, bookmarks, post_tags.
    • ON DELETE CASCADE for children that can't outlive their parent (comments, likes, post_tags, bookmarks).
  • drizzle.config.ts + src/db/migrate.ts (run migrations, called on boot from index.ts). Scripts: db:generate, db:migrate.
  • ULID helper in src/lib/ids.ts.
②

Acceptance

  • bun run db:generate produces migration SQL; db:migrate applies it.
  • Booting the app creates ./data/app.db with all tables.
  • FKs enforced (a manual insert with a bad FK fails).
  • GET /health still works (no regression).
③

Bruno tests

No new endpoints → no new Bruno folder. Just confirm 01-health still passes after wiring the DB + boot migrations.
