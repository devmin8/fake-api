Likes & bookmarks

Polymorphic likes over posts and comments with denormalized counts, plus per-user bookmarks — all through the services layer.

①

Scope

  • src/services/likes.ts — like/unlike over (target_type, target_id) with the uniqueness guard (no double-like → 409/idempotent, pick one and document). Updates like_count on the target in the same transaction.
  • Bookmark logic (in services/posts.ts or a small services/bookmarks.ts).
  • Routes:
    • POST/DELETE /api/posts/:id/like AUTH.
    • POST/DELETE /api/comments/:id/like AUTH.
    • POST/DELETE /api/posts/:id/bookmark AUTH.
    • GET /api/bookmarks AUTH — your bookmarked posts (paginated).
  • Broadcast seam left for task-011 (like → post.liked + new count).
②

Acceptance

  • Like a post → like_count +1; unlike → −1; double-like is guarded.
  • Like a comment behaves the same on the comment's count.
  • Bookmark/un-bookmark toggles; GET /bookmarks reflects it.
  • All four endpoints 401 unauthenticated.
  • Counts never drift below 0 or double-count.
③

Bruno tests — 07-likes-bookmarks/

Auth required. Reuse a postId/commentId (capture fresh if not in env). Chain: read count → like → assert count +1 → like again → assert guard → unlike → assert count back. Bookmark → appears in GET /bookmarks → un-bookmark → gone. Add 401 cases.
